# Tavern Social 群聊功能体验报告 - 第 3 轮

测试时间：2026-06-08  
测试入口：http://127.0.0.1:8088/  
测试重点：最终回归、生成等待反馈、手机一级群聊页体验

## 本轮测试路径

1. 重新构建页面并刷新。
2. 手机宽度 390px 下进入底部“群聊”。
3. 检查左上角是否只保留发言身份选择器，不再出现多余返回箭头。
4. 点击“主动续聊”，在生成过程中检查消息区提示、按钮禁用状态和横向溢出。
5. 等待生成结束，确认新增消息、提示消失、按钮恢复可用。
6. 回到桌面宽度，检查群聊页布局、身份选择、消息列表和发送按钮标签。
7. 执行完整自动化测试集合。

## 好的部分

- 手机群聊页现在更像一级页面：左上角直接是“发言身份”，没有多余返回箭头。
- 生成过程中消息区会显示“角色正在组织下一句…”，用户能知道系统正在工作。
- 生成期间相关按钮会禁用，结束后恢复，避免重复点击造成多条并发生成。
- 主动续聊生成成功后，会新增带“主动生成”标记的角色消息。
- 桌面端和手机端都没有横向溢出，控制台无错误。
- 完整测试集合通过，覆盖核心迁移、群聊、主动消息、动态、事件、状态简报、备份、导出、写卡、模型列表、提示词预设、PNG 角色卡等。

## 仍可改进的部分

- 手机群聊管理区仍然偏重。现在已经可用，但更理想的形态是把“群设置/成员/生成控制”收进一个可展开面板，只把身份选择和消息流保持常驻。
- 历史消息里的已移出成员仍正常显示，但没有额外说明“这是历史发言”。目前不影响使用，但长群聊里可以考虑加弱提示。
- 群聊目前没有单条消息撤回/删除入口。如果未来要把群聊作为长期主场，这会是自然的下一步。

## 已根据本轮报告完成的修改

- 移除手机群聊页的多余返回箭头。
- 群聊生成开始后立即刷新 UI，显示消息区内的生成中提示。
- 新增 `.group-generation-hint` 样式，让等待状态清楚但不喧宾夺主。

## 最终验证结果

- `pnpm typecheck:independent` 通过。
- `pnpm test:independent` 通过。
- `pnpm build:dev` 通过。
- 浏览器桌面端 1280x720 通过：无横向溢出，无控制台错误。
- 浏览器手机端 390x844 通过：无横向溢出，无控制台错误，底部 7 个导航入口正常。
