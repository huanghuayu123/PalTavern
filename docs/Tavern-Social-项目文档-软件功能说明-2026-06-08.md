# Tavern Social 项目文档：软件功能说明

更新日期：2026-06-08

适用项目：Tavern Social

适用版本：当前仓库版本，`0.1.0`

文档用途：向用户、测试者、协作者说明这个软件是什么、能做什么、适合怎样使用，以及当前版本的边界。

## 1. 一句话介绍

Tavern Social 是一个面向 SillyTavern 角色卡用户的独立角色社交软件。它把原本只能在酒馆聊天窗口里出现的角色，搬进一个类似手机通讯和社交应用的环境里，让角色拥有私聊、动态、评论、世界事件、关系状态、时间线记忆和主动联系节奏。

它不是一个普通聊天框，也不只是一个角色卡导入器。它更像是一个“角色生活空间”：用户可以导入或创建角色，让角色在同一个世界里聊天、发动态、回应事件、记住关系变化，并在用户不主动输入时也保留一种持续存在感。

## 2. 产品定位

Tavern Social 的核心目标，是让 AI 角色不再只是“等待用户输入后回复的一张卡”，而是变成可以长期相处、会积累生活痕迹、会因为关系变化产生不同反应的联系人。

它关注的不是单次对话爽感，而是长期陪伴感和世界连续性：

- 角色有独立私聊窗口。
- 角色有自己的关系状态和好感度。
- 角色可以主动给用户发消息。
- 角色可以发动态，像朋友圈或社交平台一样留下近况。
- 角色之间可以围绕动态和事件产生低频互动。
- 世界里发生过的事会沉淀到时间线。
- 重要事件可以影响关系，也可以撤销影响。
- 用户可以把角色卡导入、导出、备份、迁移。

## 3. 适合谁使用

Tavern Social 适合以下用户：

- 已经在使用 SillyTavern，并拥有自己喜欢的角色卡。
- 希望角色像通讯录联系人一样存在，而不是只在一个聊天页面里出现。
- 希望多个角色共存在同一个世界里，有动态、事件和共同记忆。
- 希望角色会主动联系自己，但又不想被刷屏。
- 希望保留关系、好感、时间线和事件结果。
- 希望把角色卡制作、导入、聊天、导出整合在一个独立应用里。
- 希望在 Windows、Android 或本地网页中运行一个独立角色社交应用。

它不太适合以下场景：

- 只想要最简单的一问一答聊天框。
- 需要多人实时群聊、语音通话或云端账号同步。
- 需要正式商店发布级别的 Android / iOS 应用。
- 希望应用关闭后仍然百分百实时运行所有后台任务。

## 4. 当前可运行形态

当前项目支持三种运行方式。

### 4.1 本地 Web 应用

可以在本地浏览器中打开 Tavern Social，体验完整的主要界面和功能。

适合：

- 快速测试功能。
- 本地开发调试。
- 不安装桌面或手机应用时临时使用。

限制：

- 浏览器页面关闭后，不会继续执行主动消息、自动动态和后台检查。

### 4.2 Windows 桌面应用

项目包含 Electron 桌面应用能力。

当前桌面端目标：

- 像普通桌面软件一样打开。
- 关闭窗口后可继续在托盘运行。
- 支持后台检查主动消息和自动动态。
- 可打包成 Windows 安装程序。

适合：

- 长时间挂着使用。
- 希望角色在电脑上持续保持联系感。

### 4.3 Android 应用

项目包含 Capacitor Android 工程，并且可以打包 debug APK。

当前 Android 能力：

- 以原生 Android 壳运行 Tavern Social。
- 申请通知权限。
- 使用系统通知显示角色主动消息。
- 通过 WorkManager 做后台检查。
- 可构建 debug APK 安装测试。

当前已打包产物示例：

- `TavernSocial-debug-20260608.apk`
- `android/app/build/outputs/apk/debug/app-debug.apk`

限制：

- 当前是 debug APK，不是正式签名发布包。
- Android 后台任务受系统、省电策略和厂商限制影响，不能保证像即时通讯软件一样实时。

## 5. 软件主界面能做什么

Tavern Social 的主界面围绕几个核心页面展开：

- 消息：和角色私聊。
- 联系人：管理当前世界里的角色。
- 动态：发布用户或角色动态，查看评论。
- 事件：记录或生成世界事件，选择后续结果。
- 时间线：查看世界里沉淀下来的重要记录。
- 设置：配置模型、世界、角色、关系、主动消息、提示词、备份等。

桌面端会使用更宽的左右布局，移动端会更接近手机应用，使用底部导航切换页面。

## 6. 角色卡导入与管理

### 6.1 导入 SillyTavern 角色卡

Tavern Social 可以导入 SillyTavern 角色卡，让角色进入当前世界。

支持的导入来源包括：

- SillyTavern JSON 角色卡。
- 带角色卡元数据的 PNG 角色卡。
- `chara` PNG 元数据。
- `ccv3` PNG 元数据。
- 包含 `character_book` 的世界书数据。
- 角色头像、名称、描述、性格、场景、开场白、标签等信息。

导入后，角色会获得独立数据：

- 独立私聊记录。
- 独立关系状态。
- 独立好感度。
- 独立主动消息设置。
- 独立自动动态设置。
- 独立世界事件参与记录。
- 独立表情包。
- 独立未读状态。

### 6.2 多角色识别导入

当一张卡里可能包含多个角色时，软件会尝试识别其中的角色候选。

用户可以选择：

- 按整张卡导入。
- 导入全部识别角色。
- 只导入选中的角色。

这对大型世界观卡、乐队卡、宿舍卡、班级卡或多 NPC 卡比较有用。

### 6.3 角色资料编辑

用户可以在设置中编辑角色资料，包括：

- 角色卡名。
- 角色背景故事备注。
- 角色世界书。
- 关系摘要。
- 好感度。
- 关系阶段。
- 头像。
- 表情包。

这些内容会参与模型生成，让角色回复不只依赖原始卡片。

### 6.4 导出角色卡

软件可以把当前角色导出为 SillyTavern V2 JSON 角色卡。

导出时会保留：

- 当前角色基础资料。
- 角色描述、性格、场景等字段。
- 世界书。
- Tavern Social 关系扩展信息。
- 原卡中的部分未知字段。

不会写入导出卡的内容包括：

- 用户 API Key。
- 模型配置。
- 私聊记录。
- 私人备份数据。
- 当前设备上的敏感设置。

## 7. Character Studio 写卡工具

Tavern Social 内置写角色卡工具，帮助用户从一个想法逐步写成可聊天角色。

### 7.1 两种写卡模式

简单版：

- 角色构想。
- 外貌。
- 性格。
- 爱好。
- 预览。

适合快速建立角色。

复杂版：

- 角色构想。
- 外貌。
- 爱好。
- 性格调色盘。
- 二次解释。
- 预览。

适合长期使用的角色，尤其适合避免模型误解角色性格。

### 7.2 导师式写卡

写卡工具可以让模型扮演“角色卡创作导师”。

它可以：

- 根据用户输入继续提问。
- 一次只问关键问题。
- 把零散素材整理成候选稿。
- 帮用户生成角色开场白。
- 在用户确认前不覆盖正文。

### 7.3 草稿与导出

写卡过程会自动保存草稿。

完成后可以：

- 仅保存草稿。
- 导出酒馆角色卡。
- 直接创建为 Tavern Social 联系人。

注意：写卡工具中的 `first_mes` 会写入导出的 SillyTavern 角色卡；但创建为 Tavern Social 联系人后，私聊会根据角色设定生成更自然的手机聊天开场，不会直接复制这段 `first_mes`。

## 8. 私聊功能

### 8.1 独立私聊窗口

每个角色都有自己的私聊窗口。用户可以像使用聊天软件一样给角色发送消息。

模型回复时会参考：

- 角色卡设定。
- 世界书。
- 当前世界设定。
- 用户人设。
- 当前关系阶段。
- 好感度。
- 关系摘要。
- 最近聊天。
- 最近世界事件。
- 时间线记忆。
- 角色状态摘要。
- 可用表情包。
- 当前时间模式。

### 8.2 短消息与长消息模式

软件支持两种回复习惯。

短消息模式：

- 用户可以连续发几条短消息。
- 之后手动点生成，让角色一次性回复。
- 更像真实即时通讯。

长消息模式：

- 用户发送一条完整消息后，角色自动开始回复。
- 更接近普通 AI 聊天应用。

### 8.3 自然聊天风格

Tavern Social 会尽量让模型像手机聊天一样回复，而不是每次都写长篇舞台剧。

目标风格：

- 可以拆成多条短消息。
- 避免大段旁白。
- 避免替用户行动。
- 避免替用户说话。
- 根据角色性格决定语气。
- 可以使用表情包标记。

### 8.4 重新生成和多版本回复

聊天消息支持变体概念。模型回复可以保留不同版本，用户可以在不完全丢掉上下文的前提下调整回复。

### 8.5 未读状态

角色主动发消息后，联系人列表会显示未读状态。

用户打开对应对话后，未读会被清理。

## 9. 模型连接

Tavern Social 本身不内置大模型，需要用户配置兼容 OpenAI Chat Completions 的接口。

### 9.1 支持的配置

可以配置：

- 模型厂商。
- API 地址。
- API Key。
- 模型名称。
- 温度。
- 每日请求预算。

当前模型类型包括：

- DeepSeek。
- 自定义兼容接口。

### 9.2 读取模型列表

软件可以尝试读取服务商提供的可用模型列表，用户再从列表中选择。

### 9.3 本地代理与直连

请求会优先尝试通过本地代理，代理不可用时再尝试直接连接模型接口。

常见接口地址会自动补全 `/v1/chat/completions` 这类路径。

### 9.4 请求预算

每日请求预算会限制：

- 私聊回复。
- 主动消息。
- 自动动态。
- 动态评论。
- 世界事件生成。
- 角色状态刷新。
- 今日简报生成。

这能避免自动功能在后台无限消耗模型额度。

## 10. 多世界系统

Tavern Social 支持创建多个独立世界。

每个世界可以拥有：

- 独立角色。
- 独立私聊。
- 独立动态。
- 独立事件。
- 独立时间线。
- 独立今日简报。
- 独立用户人设。
- 独立世界说明。

用户可以：

- 创建新世界。
- 切换当前世界。
- 编辑世界名称。
- 编辑世界说明。
- 设置当前世界中的用户人设。
- 删除非默认世界。

世界隔离很重要：一个世界中的角色、动态、事件和时间线不会自动串到另一个世界。

## 11. 用户人设

用户可以为当前世界设置“我是谁”。

用户人设会影响：

- 角色如何称呼用户。
- 角色如何理解用户和自己的关系。
- 私聊回复。
- 动态评论。
- 世界事件上下文。
- 主动消息触发语气。

这让同一个角色卡在不同世界中可以拥有不同的相处关系。

## 12. 时间系统

软件支持两种时间模式。

### 12.1 真实系统时间

角色会根据当前设备的真实日期、星期和时间理解早晚、作息和当天状态。

适合日常陪伴。

### 12.2 虚拟时间

用户可以手动设置角色世界里的手机时间。

适合：

- 固定在某个剧情时间。
- 模拟深夜、清晨、放学后等氛围。
- 不希望现实时间影响故事。

时间信息会写入模型提示词，影响聊天、动态和事件生成。

## 13. 关系系统

每个角色都有独立关系状态。

### 13.1 关系阶段

当前关系阶段包括：

- 陌生。
- 熟悉。
- 亲近。
- 亲密。
- 紧张。

关系阶段会影响角色的语气、主动程度、动态互动和事件反应。

### 13.2 好感度

每个角色有一个数值化的好感度，范围通常为 `-100` 到 `100`。

好感度可以：

- 手动调整。
- 被世界事件影响。
- 被关系更新记录进时间线。
- 参与模型上下文。

### 13.3 关系摘要

用户可以写一段关系摘要，例如：

- 你们是如何认识的。
- 最近关系发生了什么变化。
- 当前有哪些未解决的矛盾。
- 角色对用户大概是什么态度。

这比单纯数值更重要，因为模型会根据这段摘要理解长期关系。

## 14. 主动消息

主动消息是 Tavern Social 的核心能力之一。角色不需要一直等用户先说话，可以在合适的时间主动联系用户。

### 14.1 可以配置的内容

每个角色可以单独设置：

- 是否启用主动消息。
- 最短随机间隔。
- 最长随机间隔。
- 硬性最大间隔。
- 每日发送上限。
- 安静时段。
- 是否允许后台通知。
- 通知隐私模式。

### 14.2 主动消息触发原因

角色主动联系可能来自：

- 用户太久没有回复。
- 最近发生了世界事件。
- 关系阶段变化。
- 角色状态倾向。
- 当前时间适合联系。
- 自动调度到达检查点。

软件会保存可读理由，让用户知道“为什么这个角色现在来找我”。

### 14.3 性格化降频

如果用户连续不回复，软件不会让所有角色一直追着发消息。

角色会进入不同节奏：

- 正常节奏。
- 试探联系。
- 等待回应。
- 降频冷却。
- 暂时沉默。

这让主动消息更像一个有边界的人，而不是机械定时任务。

### 14.4 不补发原则

如果应用关闭、后台任务被系统拦截或错过检查时间，软件不会在下次打开时集中补发一堆旧消息。

这样可以避免用户被“过期消息”刷屏。

## 15. 通知功能

软件支持通知能力。

可以：

- 申请通知权限。
- 发送测试通知。
- 为主动消息开启或关闭后台通知。
- 设置通知隐私。

通知隐私模式包括：

- 显示完整内容。
- 只提示收到新消息。
- 隐藏角色名称。

Windows 桌面端和 Android 端都围绕后台存在感做了准备，但实际可靠性会受到系统环境影响。

## 16. 动态系统

动态系统类似一个角色朋友圈或世界社交流。

### 16.1 用户发布动态

用户可以以自己的身份发布动态。

可以选择可见范围：

- 公开。
- 好友可见。
- 指定角色可见。
- 屏蔽角色。
- 仅自己。

用户动态可以触发角色评论，但只有可见的角色才能参与。

### 16.2 角色发布动态

用户可以让当前角色生成并发布一条动态。

生成时会参考：

- 当前时间。
- 角色设定。
- 世界说明。
- 近期聊天。
- 近期事件。
- 关系状态。
- 近期已发动态。
- 角色自己的生活倾向。

角色动态不一定总是围绕用户，也可以只是记录角色自己的生活片段。

### 16.3 自动动态

每个角色可以单独设置自动动态。

可以配置：

- 是否启用。
- 最短和最长随机间隔。
- 每日上限。
- 安静时段。

自动动态也遵守不补发原则。

### 16.4 动态评论

动态支持评论。

可以：

- 用户评论角色动态。
- 让角色评论用户动态。
- 让角色回复动态下的新评论。
- 避免重复评论。
- 删除动态。

评论会根据角色性格、关系、可见范围和最近上下文生成。

## 17. 动态可见范围

动态不是默认所有角色都知道。可见范围决定谁能看到，谁能评论，谁能在模型上下文里引用它。

当前可见模式：

- 公开：当前世界角色都可见。
- 好友可见：关系达到一定阶段的角色可见。
- 指定角色：只有选中的角色可见。
- 屏蔽角色：公开但排除某些角色。
- 仅自己：作为私人状态记录。

可见范围会影响：

- 评论生成。
- 角色之间互动。
- 时间线上下文。
- 后续聊天是否能提到这条动态。

## 18. 角色之间的互动

Tavern Social 不只让角色和用户互动，也支持角色之间产生低频互动。

当前互动形式包括：

- 角色评论其他可见角色的动态。
- 多个角色共同参与世界事件。
- 角色互动写入时间线。
- 互动原因被保存下来。

软件会限制频率，避免角色之间刷屏。

## 19. 世界事件系统

事件系统用于记录当前世界发生的生活片段、突发新闻或关系事件。

### 19.1 手动记录事件

用户可以手动写一件事，例如：

- 某个角色遇到问题。
- 两个角色在某地碰面。
- 用户做了某个会影响关系的选择。
- 世界里发生了一条新闻。

手动事件可以设置：

- 标题。
- 正文。
- 关联角色。
- 建议关系影响。

### 19.2 模型生成事件

软件可以让模型生成当前世界的岛上新闻或角色身边事件。

生成时会参考：

- 当前世界。
- 当前角色。
- 近期聊天。
- 动态。
- 关系状态。
- 时间线。

### 19.3 事件分支

事件可以提供多个处理选择。

用户可以选择：

- 顺势参与。
- 留作记录。
- 轻轻吐槽。
- 其他由事件生成的分支。

选择后，模型可以生成事件结果。

### 19.4 手写事件结果

用户也可以不让模型决定，而是自己手写这件事最后怎样收场。

这适合用户想控制剧情走向的时候。

### 19.5 事件影响关系

事件可以影响角色好感度和关系摘要。

例如：

- 某个选择让角色更亲近。
- 某个冲突让关系变紧张。
- 某个共同经历被写进近期记忆。

### 19.6 删除与撤销影响

删除事件前，软件会确认是否删除。

如果事件已经产生关系影响，软件会询问是否同时撤销这些影响。

这能防止一次不满意的模型输出污染长期关系。

## 20. 时间线系统

时间线是 Tavern Social 的共同记忆层。它记录“这个世界发生过什么”。

时间线可以包含：

- 聊天中被手动标记的重要消息。
- 动态发布。
- 动态评论。
- 世界事件创建、结算、删除。
- 关系变化。
- 主动消息。
- 今日简报。
- 角色状态摘要。
- 角色之间互动。
- 用户手动记录。

时间线的作用：

- 让用户回顾世界历史。
- 给模型提供近期上下文。
- 避免角色忘记刚发生的重要事。
- 支持撤销影响。
- 让长期角色关系更连续。

用户可以手动添加时间线记录，也可以从聊天中把某句话记入世界时间线。

## 21. 今日简报

今日简报会总结当前世界当天值得注意的变化。

它可以包含：

- 未读消息。
- 主动消息。
- 新动态。
- 重要评论。
- 待处理事件。
- 已结算事件。
- 关系变化。
- 建议关注的角色。

今日简报适合用户每天打开软件时快速知道：“这个世界最近发生了什么？”

简报会写入时间线，方便以后回看。

## 22. 角色状态摘要

角色状态摘要用于回答：“这个角色最近怎么样？”

状态摘要可以包含：

- 当前心情。
- 与用户的关系阶段。
- 好感度。
- 关系摘要。
- 最近重要记忆。
- 未解决事项。
- 下一步倾向。
- 活跃来源。

状态摘要可以由规则生成，也可以由模型刷新。

它的价值是让角色不只是有静态设定，而是有“当前状态”。

## 23. 影响记录与回滚

长期角色应用最怕一次错误输出影响后续全部剧情。Tavern Social 因此引入影响记录。

可记录和回滚的对象包括：

- 关系好感度变化。
- 关系摘要变化。
- 时间线条目。
- 主动消息影响。
- 世界事件影响。
- 角色状态摘要。

当影响被撤销后，相关内容不会继续进入模型上下文。

这让用户可以更放心地尝试自动剧情，而不用担心一次错误生成毁掉长期档案。

## 24. 表情包系统

Tavern Social 支持表情包。

当前有三类：

- 角色专属表情包：只给指定角色使用。
- 通用表情包：所有角色都可以使用。
- 用户表情包：由用户自己发送和管理。

支持导入：

- PNG。
- JPG / JPEG。
- WebP。

导入时可以给图片写备注。

备注的作用：

- 帮模型理解表情包表达什么。
- 让角色在合适语境下选择表情。
- 方便用户管理图库。

角色回复时可以引用自己可用的表情包。

## 25. 提示词预设

软件支持导入和使用提示词预设。

当前可管理：

- 预设名称。
- 提示词条目。
- 启用 / 禁用状态。
- 排序。
- 系统提示词标记。
- Regex 脚本。
- 预设参数摘要。

用户可以：

- 导入提示词预设。
- 切换当前聊天预设。
- 开启或关闭私聊预设。
- 单独调整预设中的提示词开关。
- 恢复默认开关。
- 删除预设。

这让 Tavern Social 可以沿用部分酒馆用户已有的提示词工作流。

## 26. 本地备份与恢复

Tavern Social 的数据主要保存在本机。

可备份内容包括：

- 世界。
- 角色。
- 角色卡草稿。
- 私聊记录。
- 动态与评论。
- 世界事件。
- 时间线。
- 影响记录。
- 角色互动。
- 角色状态。
- 今日简报。
- 表情包。
- 用户人设。
- 模型配置。
- 提示词预设。
- 主动消息与自动动态设置。

用户可以导出完整备份 JSON，也可以导入备份恢复。

注意：完整备份可能包含 API Key 等敏感信息，应当妥善保存，不建议随便发给别人。

## 27. 数据隐私

当前版本以本地存储为主。

这意味着：

- 没有内置云端账号。
- 没有多设备自动同步。
- 聊天、角色、关系和备份默认在本机。
- 模型请求会发送到用户配置的 API 服务商。
- API Key 保存在当前设备的本地配置中。

对用户来说，这比云端产品更可控，但也意味着换设备时需要自己导出和导入备份。

## 28. 典型使用流程

### 28.1 第一次使用

1. 打开 Tavern Social。
2. 进入设置，填写模型 API 地址、API Key 和模型名称。
3. 创建一个世界，或者使用默认世界。
4. 设置用户人设。
5. 导入一个 SillyTavern 角色卡，或用 Character Studio 写一张新卡。
6. 进入联系人，选择角色开始私聊。
7. 根据需要设置关系、主动消息、动态、事件和通知。

### 28.2 日常使用

1. 打开软件，看今日简报。
2. 查看是否有未读消息。
3. 和角色聊天。
4. 发布一条动态，看看角色是否评论。
5. 让角色发一条自己的动态。
6. 查看事件页面，处理世界里发生的事。
7. 把重要聊天或事件记入时间线。
8. 不满意的事件影响可以撤销。

### 28.3 长期使用

1. 定期备份。
2. 给角色补充关系摘要。
3. 调整主动消息频率。
4. 管理动态可见范围。
5. 用时间线维护共同记忆。
6. 用今日简报回顾世界变化。
7. 需要迁移时导出备份或角色卡。

## 29. 它和 SillyTavern 的关系

Tavern Social 并不是替代 SillyTavern 的全部功能。

它更像是 SillyTavern 角色卡的独立社交外壳：

- 可以导入 SillyTavern 角色卡。
- 可以导出 SillyTavern V2 角色卡。
- 可以读取角色卡世界书。
- 可以使用兼容 OpenAI 的模型接口。
- 可以借鉴酒馆提示词预设工作流。

但它提供的是另一种体验：从“酒馆聊天页面”转向“手机联系人和社交世界”。

## 30. 当前还不支持什么

当前版本还不支持或未正式发布：

- iOS 原生应用。
- Android 正式签名 APK / AAB。
- 应用商店发布。
- 云端账号。
- 多设备自动同步。
- 真正实时的服务器推送。
- 群聊。
- 语音消息。
- 语音通话。
- 普通图片附件消息。
- 端到端加密。
- 服务端永久存储。
- 完整商业发布所需的隐私政策、品牌资源和签名流程。

## 33. 对外介绍短文

Tavern Social 是一个让 SillyTavern 角色卡变成独立联系人的角色社交应用。你可以导入或创建角色，和他们私聊，让他们发动态、评论、参与世界事件，并通过关系、好感度、时间线和今日简报沉淀长期记忆。它支持兼容 OpenAI 的模型接口、本地备份、角色卡导出、表情包、提示词预设，以及 Windows、Android 和本地 Web 运行形态。它的目标不是做一个普通 AI 聊天框，而是让角色像生活在同一个手机社交世界里一样，持续留下互动和关系变化。
