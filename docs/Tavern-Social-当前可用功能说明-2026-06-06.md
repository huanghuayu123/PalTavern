# Tavern Social 当前可用功能说明

更新日期：2026-06-06

适用版本：`0.1.0`

当前代码分支：`codex/tavern-social-snapshot`

## 一、产品定位

Tavern Social 是一个独立运行的角色社交与聊天应用，不依赖 SillyTavern 页面持续打开。

它可以导入 SillyTavern 角色卡，让不同角色拥有各自的私聊、关系状态、主动联系节奏、社交动态、世界事件和表情包，并通过兼容 OpenAI 接口的模型生成内容。

当前可运行形态：

- Windows 桌面应用
- Android 调试应用
- 本地 Web 页面

## 二、开始使用

1. 安装并打开 Tavern Social。
2. 在“联系人”或“设置 -> 世界与角色”中导入角色卡。
3. 在“设置 -> 模型连接”中填写 API 地址和模型名称。
4. 选择角色后开始私聊。
5. 根据需要设置关系、主动消息、自动动态、通知和表情包。

模型服务需要兼容 OpenAI Chat Completions 接口。

## 三、角色卡与角色管理

### 3.1 导入角色卡

当前支持：

- SillyTavern JSON 角色卡
- 带有角色卡元数据的 PNG 角色卡
- `chara` 和 `ccv3` PNG 元数据
- 角色名称、描述、性格、场景、开场信息、标签和世界书
- `character_book` 世界书数据
- PNG 角色头像

每个导入角色会获得独立的：

- 私聊会话
- 关系状态
- 主动消息设置
- 自动动态设置
- 角色专属表情包
- 未读消息状态

### 3.2 自定义头像

可以为角色上传新的头像，支持：

- PNG
- JPG / JPEG
- WebP

图片会在本地压缩处理，最长边限制为 `640px`。

### 3.3 导出角色卡

可以把当前角色导出为 SillyTavern V2 JSON 角色卡。

导出内容会保留原卡未知字段，并更新当前角色资料、世界书和 Tavern Social 关系扩展。API Key、模型设置和聊天记录不会写入角色卡。

## 四、世界与多角色空间

当前支持创建多个独立世界。

每个世界分别保存：

- 角色
- 私聊
- 动态
- 世界事件
- 关系状态

可以：

- 创建和切换世界
- 设置世界名称和说明
- 删除非默认世界
- 在同一世界中管理多个角色
- 搜索当前世界中的联系人

删除世界会同时删除该世界中的角色、聊天、动态和事件。

## 五、AI 私聊

### 5.1 独立聊天

每个角色拥有独立私聊历史。发送消息后，模型会结合以下内容回复：

- 角色描述和性格
- 当前场景
- 世界书
- 当前关系阶段和亲密度
- 关系摘要
- 最近聊天
- 当前世界的近期事件
- 可使用的角色及通用表情包

### 5.2 自动生成开场消息

首次打开空会话时，可以根据角色设定和当前世界重新生成一条自然私信式开场。

导入角色卡中原有的 `first_mes` 不会被直接复制为 Tavern Social 的第一条聊天消息，避免舞台叙事格式直接进入手机私聊。

### 5.3 网络聊天输出格式

模型被要求使用自然的即时通讯表达：

- 可以拆成多条短消息
- 避免长篇舞台动作描写
- 避免代替用户发言
- 可以按角色性格决定语气和消息长度
- 可以输出已授权的角色或通用表情包

### 5.4 未读消息

联系人列表会显示未读状态。打开对应会话后会自动标记为已读。

## 六、模型连接

在“设置 -> 模型连接”中可以配置：

- API 地址
- API Key
- 模型名称
- 温度
- 每日请求预算

程序会自动补全常见的 `/v1/chat/completions` 路径，并优先通过本地代理请求；代理不可用时会尝试直接连接模型接口。

每日请求预算同时约束聊天、主动消息、动态、评论和事件生成。

## 七、关系系统

每个角色可以独立设置：

- 关系阶段
- 亲密度，范围为 `-100` 到 `100`
- 关系摘要

关系阶段包括：

- 陌生
- 熟悉
- 亲近
- 亲密
- 紧张

关系数据会参与：

- 普通聊天提示
- 主动消息文风和频率
- 角色动态
- 动态评论
- 世界事件影响

关系管理对象与当前聊天对象互相独立，切换设置中的角色不会切换正在聊天的联系人。

## 八、主动消息

角色可以在用户没有主动发送消息时，自行生成并发送私聊消息。

可以为每个角色设置：

- 是否启用
- 最短随机间隔
- 最长随机间隔
- 硬性最大间隔
- 每日发送上限
- 安静时段
- 是否允许后台通知
- 通知隐私

### 8.1 性格化降频

当用户连续不回复时，系统会根据角色性格和关系状态逐步降低联系频率。

当前区分的节奏倾向包括：

- 黏人或主动型
- 克制或疏离型
- 敏感或试探型
- 普通平衡型

用户重新回复后，程序不会擅自恢复原频率，而是让用户选择：

- 恢复正常频率
- 保持当前降频节奏

### 8.2 不补发原则

应用关闭、系统阻止后台任务或错过计划时间时，不会在下次打开后集中补发旧消息。

## 九、角色动态与评论

### 9.1 用户动态

用户可以输入并发布自己的动态，也可以删除动态。

### 9.2 角色动态

可以让当前角色调用模型发布一条动态。生成时会参考：

- 当前日期、星期和时段
- 角色身份、职业或学业
- 角色性格和作息
- 当前世界
- 近期事件
- 关系状态
- 最近聊天
- 最近已经发布的动态

角色会根据情境选择发布手机动态，或记录当前生活状态，不会强制每次都写成朋友圈文案。

### 9.3 自动动态

每个角色可以独立设置自动动态：

- 是否启用
- 最短和最长随机间隔
- 每日动态上限
- 安静时段

错过的自动动态不会补发。

### 9.4 动态评论

当前支持：

- 用户评论动态
- 让当前角色评论用户动态
- 角色根据关系、性格和最近聊天生成评论
- 避免重复已有评论
- 删除动态

不同世界的动态互相隔离。

## 十、世界事件

可以在当前世界中：

- 手动创建事件
- 让当前角色生成生活事件
- 指定参与角色
- 设置亲密度影响
- 标记事件结束
- 删除事件

近期世界事件会进入角色聊天、动态和其他模型生成上下文，使角色行为保持一定连续性。

## 十一、表情包

当前有三类表情包：

1. 角色专属表情包：只供指定角色使用。
2. 通用表情包：所有角色都可以使用。
3. 用户表情包：只由用户发送。

支持批量导入：

- PNG
- JPG / JPEG
- WebP

每类图库最多保存 `48` 个表情包。聊天面板会合并显示用户表情包和通用表情包，最多 `96` 个。

角色可以根据提示中提供的表情包名称，在模型回复中选择角色专属或通用表情包。

## 十二、通知

可以：

- 申请通知权限
- 发送测试通知
- 为主动消息开启或关闭后台通知
- 设置每个角色的通知隐私

通知隐私模式：

- 完整内容
- 只提示收到新消息
- 隐藏角色名称

## 十三、数据保存、备份与恢复

应用数据默认保存在本地，包括：

- 世界和角色
- 聊天记录
- 关系状态
- 动态与评论
- 世界事件
- 表情包
- 模型配置
- 主动消息和自动动态设置

可以导出完整备份 JSON，也可以导入备份并恢复本地状态。

备份格式标识为 `tavern-social-backup-v1`。

注意：完整备份包含本地模型配置，其中可能包括 API Key，应妥善保管备份文件。

## 十四、平台能力

### 14.1 Windows

当前支持：

- 独立安装程序
- 最小化到系统托盘
- 关闭窗口后继续在托盘运行
- 重复启动时唤回已有窗口
- 应用进程仍运行时继续检查主动消息和自动动态

当前安装程序：

`release/Tavern Social-0.1.0-x64.exe`

### 14.2 Android

当前支持：

- Android 调试 APK
- Capacitor 原生应用壳
- 通知权限申请
- 原生系统通知
- WorkManager 后台检查请求

当前 APK：

`android/app/build/outputs/apk/debug/app-debug.apk`

Android 后台任务由系统决定执行时间。系统省电策略、厂商限制或进程被彻底回收时，任务可能延迟或跳过。

### 14.3 Web

普通 Web 页面可以使用聊天和管理功能，但页面关闭后不会继续执行主动消息或自动动态。

## 十五、当前尚未支持

以下功能目前没有完成或没有正式发布能力：

- iOS 原生应用和后台任务
- Android 正式签名 APK / AAB
- 应用商店发布
- 云端账号和多设备同步
- 群聊
- 语音消息和语音通话
- 除表情包以外的普通图片附件消息
- 服务器端永久存储
- 端到端加密
- Windows 正式品牌图标和代码签名
- Android 被系统彻底回收后的可靠实时后台运行
