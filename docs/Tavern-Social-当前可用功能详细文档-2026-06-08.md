# Tavern Social 当前可用功能详细文档

更新日期：2026-06-08

适用范围：当前工作区中的 Tavern Social，项目版本以 `package.json` 的 `0.1.0` 为准。本文只说明当前已经能在应用里使用、能被本地状态保存、或已经接入运行流程的功能；不把规划项写成已交付能力。

## 1. 产品定位

Tavern Social 是面向 SillyTavern 角色卡用户的独立角色社交应用。它把角色卡从单一聊天窗口里拆出来，变成一个可长期使用的联系人、群聊成员和世界居民。

用户可以在一个本地世界中导入或创作角色，让角色拥有：

- 独立私聊。
- 群聊发言。
- 关系阶段、好感度和关系摘要。
- 主动消息节奏。
- 自动动态和自动事件。
- 社交动态、评论和角色间互动记录。
- 世界事件、时间线、今日简报和角色状态。
- 本地表情包、备份和角色卡导出。

它不是云服务，也不是正式账号系统。当前数据主要保存在本机；模型能力依赖用户配置兼容 OpenAI Chat Completions 的接口。

## 2. 当前可运行形态

### 2.1 本地 Web 应用

当前可以作为本地 Web 应用运行，适合测试、调试和临时使用。

可用能力：

- 完整主界面。
- 角色导入、写卡、私聊、群聊、动态、事件、时间线和设置。
- 本地状态保存。
- 本地备份 JSON 导入导出。

限制：

- 浏览器页面关闭后，主动消息、自动动态和自动事件不会继续运行。
- 浏览器通知是否可用取决于当前浏览器权限。

### 2.2 Windows 桌面应用

当前包含 Electron 桌面端运行逻辑。

可用能力：

- 以桌面窗口打开 Tavern Social。
- 关闭窗口或最小化时隐藏到系统托盘。
- 应用进程仍在运行时，调度器可以继续检查主动消息、自动动态和自动事件。
- 重复启动时唤回已有窗口。
- 桌面通知权限由 Electron 宿主处理。

限制：

- 只有应用进程仍在运行时才会继续检查。
- 用户真正退出应用后不会补发错过的主动消息或自动内容。
- 本次检查确认桌面运行逻辑、Electron 依赖和打包配置存在，但没有找到 `release/win-unpacked/TavernSocial.exe` 和 `release/Tavern Social-0.1.0-x64.exe`，因此当前不能把 `0.1.0` Windows 安装包说成已经产出。

### 2.3 Android 应用

当前包含 Capacitor Android 工程和原生后台桥。

可用能力：

- 以 Android 壳运行 Tavern Social。
- 申请系统通知权限。
- 通过原生插件发送系统通知。
- 通过 WorkManager 请求下一次后台检查。
- 通过 Android 原生保存逻辑把备份写入 `Download/TavernSocial`。
- 当前 `release/` 目录中已有 APK 产物，具体安装包以实际交付文件为准。

限制：

- Android 后台任务会受系统调度、省电策略、网络状态和厂商限制影响。
- WorkManager 只负责请求调度，不能保证像即时通讯软件一样准时。
- 如果 Android 只恢复了进程但 WebView 不在，后台 Worker 会安全跳过。
- 当前不应承诺 iOS 后台能力。

## 3. 第一次使用流程

推荐流程：

1. 打开 Tavern Social。
2. 在“联系人”或“设置 -> 世界与角色”中导入角色卡，或者点击“写角色卡”创建原创角色。
3. 在“设置 -> 模型连接”中填写 API 地址、API Key、模型名称和温度。
4. 在“设置 -> 聊天与人设”中填写用户名称、当前世界的 user 人设和聊天回复方式。
5. 选择角色进入私聊，或创建群聊。
6. 根据需要配置关系状态、主动消息、自动动态、自动事件、提示词预设、通知和表情包。
7. 使用一段时间后，在“设置 -> 数据与运行”导出完整备份 JSON。

没有配置模型时，部分手动功能仍然可用，例如创建世界、导入角色卡、编辑资料、写草稿、发布手写动态、手动记录事件、管理表情包和备份。模型生成类功能需要有效模型接口；私聊在未配置模型时会给出本地占位回复，不能视为真实角色生成。

## 4. 主界面

### 4.1 桌面布局

桌面端使用左侧联系人栏和右侧内容区。

左侧可以：

- 切换当前世界。
- 搜索联系人或群聊。
- 查看今日变化入口。
- 进入最近联系列表。
- 写角色卡。
- 导入角色卡。
- 打开设置。
- 打开群聊入口。

右侧可以在当前角色或当前世界下切换：

- 私聊。
- 动态。
- 事件。
- 时间线。
- 群聊列表或群聊窗口。

### 4.2 移动布局

移动端使用底部导航。

当前可用页面：

- 消息：最近私聊和群聊。
- 联系人：当前世界的角色列表。
- 动态：用户和角色发布的近况。
- 事件：生活线索和世界事件。
- 时间线：世界记录和今日简报。
- 设置：世界、角色、模型、关系、主动消息、通知、备份等。

进入私聊、群聊或设置详情后，底部导航会隐藏，页面会以更接近手机应用的层级返回方式运行。

## 5. 世界与用户人设

Tavern Social 支持多世界。每个世界拥有独立的角色、群聊、私聊、动态、事件、时间线、角色状态、今日简报和 user 人设。

当前可以：

- 创建新世界。
- 切换世界。
- 修改世界名称和说明。
- 删除非唯一世界。
- 为世界填写 user 人设。
- 为世界搜索现实城市。
- 保存城市位置。
- 刷新当前天气。

世界信息会参与模型生成。角色私聊、动态、事件和群聊会根据当前世界读取相关上下文。

世界天气来自 Open-Meteo。应用会保存城市、天气快照、观测时间和刷新时间；模型提示词中可以使用当前天气、温度、体感、湿度、风速和昼夜信息。天气是辅助上下文，不是角色必须提到的内容。

## 6. 角色卡导入、识别、编辑与导出

### 6.1 导入角色卡

当前支持导入：

- SillyTavern JSON 角色卡。
- 带角色卡元数据的 PNG 角色卡。
- `chara` PNG 元数据。
- `ccv3` PNG 元数据。
- 带 `character_book` 的角色卡。
- 角色头像、名称、描述、性格、场景、开场白、标签、作者备注和世界书。

导入后，角色会获得当前世界中的独立数据：

- 独立私聊会话。
- 独立关系状态。
- 独立主动消息设置。
- 独立自动动态设置。
- 独立自动事件设置。
- 独立专属表情包。
- 独立未读状态。

角色卡导入时会把描述、性格、场景整理成角色绑定的“设定世界书正文”，避免把大量设定塞进普通简介字段里。

### 6.2 多角色候选识别

如果一张卡里可能包含多个角色，应用会尝试从以下内容识别候选角色：

- 顶层卡名。
- 角色列表、人物列表、登场人物、NPC 等文本模式。
- 世界书条目。
- 结构化字段。

导入弹窗中可以选择：

- 按整张卡导入。
- 全部候选都导入。
- 只导入勾选角色。

这对乐队卡、宿舍卡、班级卡、世界观卡、多 NPC 卡比较有用。

### 6.3 角色资料编辑

当前可以编辑：

- 卡名。
- 设定世界书正文。
- 自定义头像。
- 角色专属表情包。
- 关系阶段。
- 好感度。
- 关系摘要。
- 主动消息策略。

自定义头像支持 PNG、JPG、JPEG 和 WebP。图片会在本地压缩处理，最长边限制为 640px。

### 6.4 角色删除

删除角色会移除：

- 角色本体。
- 对应私聊会话和消息。
- 角色动态。
- 角色状态。
- 群聊成员中的该角色引用。
- 事件参与者中的该角色引用。
- 草稿中与该角色的绑定关系。

删除角色不会删除其他角色，也不会删除整个世界。

### 6.5 导出酒馆角色卡

当前可以把 Tavern Social 中的角色导出为 SillyTavern V3 JSON 角色卡。

导出会保留或写入：

- `spec: chara_card_v3`。
- `spec_version: 3.0`。
- 角色基础资料。
- 角色设定世界书。
- 原始卡中的部分未知字段。
- Tavern Social 扩展信息。

导出不会写入：

- API Key。
- 模型配置。
- 私聊记录。
- 本地备份数据。
- 设备通知设置。

## 7. Character Studio 写卡工具

Tavern Social 内置写卡工具，可以从一个想法逐步写成角色卡。

### 7.1 写卡模式

简单版步骤：

- 角色构想。
- 外貌。
- 性格。
- 爱好。
- 完成预览。

复杂版步骤：

- 角色构想。
- 外貌。
- 爱好。
- 性格调色盘。
- 二次解释。
- 完成预览。

复杂版适合长期角色，尤其适合把“这个性格不等于什么”“模型容易误解什么”写清楚。

### 7.2 草稿管理

当前可以：

- 自动保存写卡输入。
- 继续草稿。
- 复制草稿。
- 删除草稿。
- 查看草稿最近更新时间。
- 保留已创建角色和草稿之间的绑定。

### 7.3 模型导师

配置模型后，写卡工具可以让模型作为角色卡创作导师：

- 针对当前步骤继续提问。
- 把零散素材整理成候选稿。
- 生成 first_mes 开场白。
- 在用户确认前不覆盖正文。

没有配置模型时，仍然可以手写所有字段并创建角色。

### 7.4 完成后的动作

预览页可以：

- 仅保存草稿。
- 导出 SillyTavern 角色卡。
- 创建为 Tavern Social 联系人。

注意：写卡工具里的 first_mes 会写入导出的酒馆角色卡；但作为 Tavern Social 联系人使用时，私聊开场会根据当前应用场景另行生成，不会直接把 first_mes 当成第一条手机私信。

## 8. 私聊功能

### 8.1 独立私聊

每个角色都有独立私聊窗口。角色消息和用户消息保存在当前世界下，不会和其他角色混在一起。

私聊模型回复会参考：

- 当前角色设定。
- 角色世界书。
- 近期私聊。
- 当前世界说明。
- user 人设。
- 当前时间或虚拟陪伴时间。
- 当前天气。
- 关系阶段、好感度和关系摘要。
- 近期世界事件。
- 世界时间线。
- 今日简报。
- 角色状态摘要。
- 可用表情包。
- 私聊提示词预设。

### 8.2 自动生成开场

首次打开空会话时，应用可以根据角色设定生成新的手机私信式开场。

开场生成规则：

- 不直接复制原角色卡 first_mes。
- 不把舞台剧式开场直接塞进手机聊天。
- 尽量写成角色自然发来的第一条私信。
- 控制在 1 到 4 个短段落内。

### 8.3 两种回复方式

短消息模式：

- 用户可以连续发送多条短消息。
- 之后点击“生成”，角色再统一回复。
- 适合模拟真实聊天中的连续输入。

长消息模式：

- 用户发送一条完整消息后，角色自动回复。
- 更接近传统 AI 聊天。

### 8.4 消息操作

当前私聊消息支持：

- 停止正在生成的回复。
- 删除消息。
- 撤回消息并保留痕迹。
- 修改用户消息并重新生成后续。
- 重新生成 AI 回复。
- 在多个回复版本之间切换。
- 引用某条消息回复。
- 把消息记入世界时间线。
- 显示未读消息数。

删除用户消息会删除该消息之后同一会话中的后续消息；撤回会保留痕迹，提示“撤回了一条消息”。

### 8.5 聊天风格保护

默认私聊提示词会要求模型：

- 输出角色会发出的消息。
- 避免括号动作描写。
- 避免星号动作。
- 避免心理旁白。
- 避免替用户行动或发言。
- 不承诺应用没有提供的能力，例如发送照片、语音、文件、定位、现实提醒、保存、上传、设置闹钟等。

模型可以用 `<msg>...</msg>` 拆成多条短消息，也可以用表情包标记选择可用表情包。

## 9. 群聊功能

### 9.1 群聊列表

当前世界可以创建多个群聊。群聊列表支持：

- 创建群聊。
- 搜索群聊名称或成员。
- 按最近发言排序。
- 显示群成员头像堆叠。
- 从消息页进入群聊。

### 9.2 群聊设置

每个群聊可以配置：

- 群名称。
- 群成员。
- 当前手写发言身份。
- 是否让每位成员回复上一条用户消息。
- 是否允许模型主动发言。

群聊可以清空聊天记录，也可以解散。清空只删当前群记录；解散会删除群聊和记录，不会删除角色卡。

### 9.3 群聊发言

当前支持：

- 用户以自己身份发言。
- 用户选择某个角色身份手写发言。
- 让指定角色说一句。
- 生成下一步。
- 让群聊继续接上一条。
- 在允许后让模型主动续聊。
- 一轮由模型判断 0 到 2 位角色自然接话。
- 开启“每位成员回复上一条”后，对用户发言生成一轮群成员回复。

群聊生成有保护：

- 默认不允许模型主动开话题，避免 token 消耗失控。
- 主动续聊必须在群聊设置里显式开启。
- 一轮最多生成有限数量的气泡。
- 角色可以输出“跳过”，表示这轮自然沉默。
- 会避免同一角色连续自说自话。

群聊消息会写入世界时间线，作为后续上下文的一部分。

## 10. 模型连接

Tavern Social 不内置大模型。用户需要提供兼容 OpenAI Chat Completions 的模型接口。

当前模型设置包括：

- 模型厂商。
- API 地址。
- API Key。
- 模型名称。
- 温度。
- 每日自动输出预算。

当前厂商选项包括：

- DeepSeek。
- 自定义兼容接口。

API 地址会自动识别常见路径：

- 聊天请求使用 `/v1/chat/completions`。
- 模型列表使用 `/v1/models`。

请求会优先尝试本地代理接口；代理不可用时会尝试直接连接模型服务。

### 10.1 模型列表

模型连接页可以尝试获取模型列表。获取成功后，用户可以从列表中选择模型，也可以手动填写模型名。

### 10.2 自动输出预算

每日自动输出预算主要约束自动或高消耗功能，例如：

- 主动消息。
- 自动动态。
- 自动事件。
- 角色自动判断是否评论动态。
- 高模拟世界互动。
- 群聊主动续聊中的部分模型判断。

普通手动聊天、手动写卡、手动编辑、手动发布动态和手动记录事件不应被理解为这个预算的主要限制对象。

## 11. 提示词预设

当前支持导入 SillyTavern 预设 JSON，并在 Tavern Social 内编辑和选择。

可用能力：

- 导入预设 JSON。
- 完整保存原始 JSON。
- 保存扩展字段。
- 保存 SPreset 数据。
- 保存正则脚本。
- 查看模型参数摘要。
- 私聊和群聊分别选择预设。
- 私聊和群聊分别开关预设。
- 编辑提示词名称、角色和内容。
- 启用或禁用单条 prompt。
- 恢复默认开关。
- 删除当前预设。
- 写入 Tavern Social 默认私聊预设。
- 写入 Tavern Social 默认群聊预设。

动态槽位由应用填充，例如聊天历史、角色设定、世界书、关系、时间线、群成员、群聊历史、当前发言角色等。

正则系统当前用于处理使用该预设的私聊或群聊模型输出，不处理动态、事件和写卡生成。

## 12. 关系系统与角色状态

### 12.1 关系状态

每个角色都有独立关系状态。

当前可设置：

- 关系阶段。
- 好感度。
- 关系摘要。

关系阶段包括：

- 陌生。
- 熟悉。
- 亲近。
- 亲密。
- 紧张。

好感度当前没有严格上限，界面进度条只显示 0 到 100，实际数值可以继续增加。

关系数据会进入：

- 私聊。
- 主动消息。
- 动态生成。
- 动态评论。
- 群聊角色上下文。
- 世界事件和结算。
- 角色状态摘要。

### 12.2 自动关系变化

当前已有两类自动关系影响：

- 私聊中每累计一定次数用户发言，会让关系更稳定并增加少量好感度。
- 世界事件结算后，会按事件结果对参与角色的关系产生影响。

这些变化会进入时间线。部分由事件或主动消息造成的影响可以通过时间线撤销。

### 12.3 角色状态

应用会为角色生成或刷新状态摘要。

状态摘要包含：

- 当前心情或近况。
- 当前关系阶段。
- 好感度。
- 关系摘要。
- 近期记忆标题。
- 未解决事项。
- 下一步倾向。
- 摘要来源。
- 更新时间。

私聊顶部会显示角色状态摘要。展开后可以看到未解决事项、关系和下一步倾向；角色面板里可以查看完整状态。

刷新角色状态会写入时间线，并记录可撤销影响。

## 13. 主动消息、自动动态和自动事件

### 13.1 主动消息

每个角色可以单独开启主动消息。

可配置项：

- 启用或关闭。
- 最小随机间隔。
- 最大随机间隔。
- 每日上限。
- 硬性最大间隔。
- 安静时段。
- 是否允许后台通知。
- 通知隐私。
- 主动消息节奏策略。

主动消息会根据关系、角色性格、未回复次数和节奏策略决定语气。

当前有未回复降频逻辑：

- 用户一直不回复时，会按角色性格逐步放慢联系。
- 黏人型放慢较轻。
- 克制型放慢明显。
- 敏感型可能先试探再沉默。
- 普通型按未回复次数逐步降频。

用户回复后，应用不会自动恢复原频率，而是提示用户选择：

- 恢复正常频率。
- 保持当前降频。

### 13.2 不补发原则

主动消息、自动动态和自动事件都遵守不补发原则：

- 应用关闭期间错过的计划不会在下次打开后集中补发。
- 应用会重新安排下一次尝试。
- 状态说明会记录“已跳过，不补发”。

### 13.3 自动动态

每个角色可以单独开启自动动态。

可配置项：

- 启用或关闭。
- 最小随机间隔。
- 最大随机间隔。
- 每日上限。
- 安静时段。

自动动态会让角色以自己的手机身份发布近况。它不是私聊回复，不会直接接用户最后一条私聊。动态会进入动态流和时间线。

### 13.4 自动事件

每个角色可以单独开启自动事件。

可配置项：

- 启用或关闭。
- 最小随机间隔。
- 最大随机间隔。
- 每日上限。
- 安静时段。

自动事件会生成当前世界里的轻量生活线索。事件需要用户选择分支或手写结果后，才会结算关系影响。

世界还有总量保护：

- 轻量模式下，世界自动事件数量更克制。
- 高模拟模式下，世界互动更活跃，但 token 消耗明显增加。

## 14. 动态与评论

### 14.1 发布动态

动态页可以发布近况。

当前可以选择发布身份：

- 用户自己。
- 当前世界中的任一角色。

如果选择角色，可以让模型生成动态草稿，再由用户决定是否发布。

动态会进入当前世界的动态流，并写入时间线。

### 14.2 可见范围

动态支持可见范围：

- 公开。
- 好友可见。
- 指定角色。
- 屏蔽角色。
- 仅自己。

好友可见当前按关系阶段判断：熟悉、亲近、亲密的角色可以看到。

可见范围会影响：

- 哪些角色能被指定评论。
- 哪些角色会在自动互动中看到动态。
- 相关动态或评论是否进入某个角色可读的时间线上下文。

### 14.3 评论

当前支持：

- 用户评论动态。
- 用户选择任一可见角色手写评论。
- 指定某个角色生成评论。
- 角色作为楼主回复评论。
- 回复指定评论。
- 删除评论。
- 删除动态。

角色评论会先判断可见性。角色之间的评论会记录为角色互动，并写入时间线。

删除评论时，相关时间线和角色互动会被撤销；删除用户评论时，后面紧跟的楼主模型回复也会一起移除，避免留下失去上下文的回复。

### 14.4 高模拟互动

开启高模拟世界互动后，自动动态可以带动：

- 可见角色判断是否评论。
- 楼主按兴趣回复其他角色评论。
- 世界互动记录增加。

该模式会明显增加模型请求和 token 消耗，默认关闭。

## 15. 生活线索与世界事件

事件页当前显示为“生活线索”。

### 15.1 创建事件

当前可以：

- 手动记录一件事。
- 选择关联角色或整个世界。
- 填写标题和正文。
- 设置建议关系影响。
- 调用模型生成一条生活线索。
- 由自动事件生成生活线索。

事件类型包括：

- 日常。
- 关系。
- 问题。
- 消息。

生成事件时，模型会被要求写成手机应用里的轻量记录，而不是小说场景或线下行动。

### 15.2 事件分支

事件可以包含 2 到 3 个手机操作按钮。用户可以选择一个分支，让模型生成后续记录和最终关系影响。

也可以：

- 手写事件结果。
- 直接记为结束。
- 删除事件。

事件结算后会：

- 标记为已结算。
- 保存选择和结果。
- 对参与角色关系产生影响。
- 写入事件时间线。
- 写入关系变化时间线。
- 记录角色间互动。

### 15.3 事件删除与撤销

删除事件会：

- 撤销事件本身的时间线来源。
- 撤销已结算事件的时间线来源。
- 撤销相关角色互动记录。
- 默认回滚该事件造成的关系影响。
- 添加一条“事件已删除”的记录，说明后续模型不再读取这条事件。

## 16. 时间线、今日简报和手动记录

### 16.1 世界时间线

时间线会把世界中值得记住的内容沉淀下来。

当前记录类型包括：

- 私聊记忆。
- 群聊记忆。
- 动态。
- 评论。
- 事件。
- 关系。
- 主动消息。
- 今日简报。
- 角色状态。
- 角色互动。
- 系统记录。
- 手动记录。

时间线记录可以显示：

- 来源类型。
- 关联角色。
- 记录时间。
- 是否进入模型上下文。
- 是否已撤销。
- 是否有可撤销影响。

### 16.2 手动记录

时间线页可以手动记下一件事。手动记录会进入当前世界时间线，并作为近期世界上下文供模型参考。

适合记录：

- 用户希望角色以后记得的事实。
- 发生在世界里的重要变化。
- 不属于某条动态或事件的背景补充。

### 16.3 撤销影响

部分时间线记录可以撤销影响。

当前撤销能力覆盖：

- 事件结算造成的关系变化。
- 主动消息写入的消息影响。
- 角色状态刷新影响。
- 与这些操作关联的时间线记录。

撤销后，记录会标记为不再进入模型上下文，并新增系统回滚记录。

### 16.4 今日简报

今日简报会总结当前世界当天的变化。

简报内容包括：

- 标题。
- 总结。
- 分段说明。
- 建议关注角色。
- 未读数量。
- 变化数量。

如果当天没有明显变化，会显示安静状态。今日简报会出现在侧边栏或移动端消息页，也会在时间线页作为顶部卡片显示。

今日简报是总结，不是可撤销事实源；模型会被要求把它作为参考摘要，而不是覆盖真实时间线。

## 17. 表情包

当前有三类表情包：

- 角色专属表情包：只供指定角色模型使用。
- 通用表情包：所有角色模型可使用，用户也能从聊天面板发送。
- 用户表情包：只显示在用户聊天面板中，不提供给角色模型。

支持格式：

- PNG。
- JPG / JPEG。
- WebP。

当前限制：

- 每类图库最多 48 个。
- 用户聊天面板会合并显示用户表情包和通用表情包，最多 96 个。

导入表情包时可以编辑：

- 名称。
- 备注。

备注会帮助没有识图能力的模型理解图片，例如“害羞捂脸”“得意挑眉”“冷淡点头”。

## 18. 通知与后台提醒

当前通知能力包括：

- 申请通知权限。
- 发送测试通知。
- 主动消息生成后发送本地通知。
- Android 使用原生通知权限。
- Web 和桌面环境使用 Notification API 或 Electron 通知能力。

主动消息通知隐私有三种：

- 完整内容：显示角色名和消息正文。
- 只提示收到新消息：显示角色名，不显示正文。
- 隐藏角色名称：只显示“有新消息”。

后台运行能力取决于平台：

- Web：页面打开时检查。
- Windows 桌面：应用驻留托盘时继续检查。
- Android：请求系统后台调度，但不保证准时。
- iOS：当前未接入原生后台任务。

## 19. 数据保存、备份与恢复

### 19.1 本地保存

当前应用状态保存在本地，状态键为 `tavern-social-state-v1`。

本地状态包括：

- 世界。
- 角色。
- 写卡草稿。
- 表情包。
- 私聊会话和消息。
- 群聊和群聊消息。
- 动态和评论。
- 世界事件。
- 时间线。
- 影响记录。
- 角色互动。
- 角色状态。
- 今日简报。
- 当前视图和当前选择。
- user 名称和 user 人设。
- 提示词预设。
- 模型配置。
- 模型使用计数。

### 19.2 导出备份

备份格式为 JSON，外层标识：

- `schema: tavern-social-backup-v1`
- `app: Tavern Social`
- `exportedAt`
- `state`

Web 和桌面环境会下载 `tavern-social-backup-<timestamp>.json`。

Android 环境会优先通过原生插件保存到下载目录下的 TavernSocial 文件夹。

### 19.3 导入备份

导入备份时，应用会读取 JSON，并通过状态归一化逻辑恢复数据。可以导入完整备份 envelope，也可以导入直接包含状态字段的 JSON。

重要提醒：

- 完整备份包含本地模型配置，可能包含 API Key。
- 备份文件应当像账号密码一样妥善保存。
- 导入备份会替换当前本地状态。

## 20. 当前明确边界

当前不应承诺以下能力：

- 云端账号。
- 多设备实时同步。
- 端到端加密云备份。
- 正式 iOS 后台任务。
- 应用完全关闭后仍实时运行。
- Android 像即时通讯软件一样准时推送。
- 语音通话、真实照片发送、文件发送、定位共享、现实提醒和闹钟。
- 角色真实执行线下动作或现实设备操作。
- 群聊无限自动对话。

模型输出也会被提示不要承诺这些能力；如果模型仍然说出类似内容，应以应用实际能力为准。

## 21. 推荐使用方式

轻量使用：

- 导入 1 到 3 个角色。
- 配好模型。
- 使用私聊和动态。
- 手动记录少量事件。
- 定期导出备份。

长期陪伴使用：

- 为每个角色补充关系摘要。
- 开启少量主动消息。
- 给亲近角色设置较低频率。
- 为世界设置 user 人设、城市和天气。
- 每天查看今日简报和时间线。
- 只给重要事件做结算，避免时间线噪音。

多角色世界使用：

- 把同一世界观角色放在一个世界里。
- 为角色建立群聊。
- 开启少量自动动态。
- 需要更活跃时再开启高模拟世界互动。
- 用事件页记录角色之间的小变化。
- 用时间线撤销不满意的事件影响。

写卡使用：

- 先用简单版快速成卡。
- 对长期角色用复杂版补“性格调色盘”和“二次解释”。
- 导出酒馆卡前检查 first_mes。
- 创建为联系人后，在关系摘要里写清 user 和角色当前处境。
