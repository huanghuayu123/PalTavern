# 独立应用构建与下载记录

日期：2026-06-05  
项目：Tavern Social  
工作区：`D:\tavern_helper_template-main`

本文档是聊天记录丢失时的恢复备份，记录本次独立应用基底、平台依赖下载、原生实现、编译修复、产物和验证结果。

## 一、产品边界

- 当前产品是独立聊天软件，不是酒馆内的小手机脚本。
- 新主线源码位于 `src/independent-chat/`。
- 旧 `src/local-phone/` 只保留为历史材料，不参与独立应用默认构建。
- 酒馆关联能力通过角色卡导入、世界书读取和 SillyTavern V2 导出实现。

## 二、已实现功能

- JSON / PNG 酒馆角色卡导入。
- 多世界、多角色、每角色独立私聊。
- OpenAI-compatible 模型地址、密钥、模型名和温度配置。
- 本地数据存储、完整备份导出与恢复。
- 角色关系、动态、世界事件和基础生活模拟。
- 主动消息的随机区间、安静时段、每日上限、最大间隔和预算限制。
- 用户未回复时按角色性格逐步降频。
- 用户回复后不自动恢复频率，等待用户选择。
- 应用重开后不补算、不补发错过的主动消息。
- 通知支持完整内容、只提示新消息、隐藏角色名。
- Windows 托盘后台驻留。
- Android WorkManager 后台尝试与原生通知。

## 四、Android 实施步骤

1. 使用 Capacitor CLI 生成 `android/` 原生工程。
2. 同步 `dist/independent-chat` 到 Android WebView。
3. 配置本机 Android SDK 路径。
4. 接受 SDK 许可并安装 Platform 36 与 Build Tools。
5. 在 `android/app/build.gradle` 加入 WorkManager。
6. 在清单中加入 Android 13+ 通知权限。
7. 新增 `TavernSocialBackgroundPlugin.java`。
8. 新增 `TavernSocialBackgroundWorker.java`。
9. 在 `MainActivity.java` 注册插件和后台事件派发。
10. Web 端通过 `platform.ts` 接入排期、权限和通知方法。
11. 首次编译发现 `onDestroy()` 权限低于 Capacitor 基类，改为 `public`。
12. 再次执行 Gradle，93 个任务完成，构建成功。

Android 的设计仍严格遵守：

- 不使用首版默认常驻前台服务。
- 系统限制时允许延迟或跳过。
- WebView 已被回收时安全跳过。
- 不在下次打开应用时补发。

## 五、Windows 实施步骤

1. 下载 Electron Windows 运行文件。
2. 将 `electron` 调整为开发依赖。
3. 使用 Electron Builder 生成 `win-unpacked`。
4. 发现根项目依赖会使安装包膨胀，新增 `desktop/package.json` 作为独立发行清单。
5. Electron Builder 的应用目录改为 `desktop/`。
6. 仅把独立服务脚本和 `dist/independent-chat` 作为运行资源打包。
7. 新增隐藏自检模式，实际启动打包后的程序、启动内置服务并加载应用界面。
8. 隐藏自检退出码为 `0`。
9. 使用 NSIS 生成可选择安装目录的 Windows 安装程序。

## 十、设置中心 UI 重构

追加日期：2026-06-05

问题：

- 旧界面采用固定三栏布局。
- 世界、模型、角色、关系、主动消息、通知、备份和运行记录全部堆在右侧栏中。
- 设置内容形成一列到底的长页面，聊天区域也被持续压缩。

调整：

1. 主界面改为“通讯录 + 内容区”两栏结构。
2. 移除常驻右侧设置栏。
3. 左侧只保留品牌、世界快速切换、角色卡导入和通讯录。
4. 新增独立设置中心，支持背景遮罩、关闭按钮和 `Esc` 关闭。
5. 设置中心拆分为六个独立分类：
   - 世界与角色
   - 模型连接
   - 关系状态
   - 主动消息
   - 通知
   - 数据与运行
6. 每次只渲染当前设置分类，不再把全部设置纵向堆叠。
7. 桌面端使用居中的大尺寸设置窗口。
8. 窄屏下设置中心自动切换为全屏页面，分类导航改为横向滚动。
9. 主界面在窄屏下保留独立的“设置”入口。

验证：

- `pnpm typecheck:independent` 通过。
- `pnpm test:independent` 全部通过。
- `pnpm build` 通过。
- 浏览器真实打开新版主界面，确认右侧长设置栏已经移除。
- 点击“设置中心”后成功显示六个设置分类。
- “模型连接”和“主动消息”分类能够独立切换。
- 关闭设置中心后正常返回聊天主界面。
- 浏览器控制台无错误或警告。
