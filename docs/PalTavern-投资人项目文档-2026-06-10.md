# PalTavern 面向投资人的项目文档

更新日期：2026-06-10

对外项目名：PalTavern

当前工程 / 历史文档名：Tavern Social

文档用途：用于投资人、合作伙伴、早期顾问和创作者生态伙伴快速理解 PalTavern 的产品方向、当前进展、商业化可能性与下一阶段融资用途。

## 1. 执行摘要

PalTavern 是一个面向 AI 角色卡用户的本地优先角色社交应用。它把 SillyTavern 等角色卡生态中的角色，从单一聊天窗口升级为可以长期存在的“社交联系人”：角色可以私聊、进入群聊、发布动态、评论、参与世界事件，并通过关系状态、时间线和今日简报沉淀长期记忆。

今天的大多数 AI 角色产品仍停留在“用户打开一个聊天框，角色被动回复”的形态。PalTavern 试图解决的是下一层体验：当用户拥有多个角色、多个世界观、长期关系和创作素材时，如何让这些角色像生活在同一个社交空间里，而不是散落在一条条孤立对话里。

PalTavern 当前已经完成可运行 MVP。项目具备本地 Web 应用、Windows 桌面应用能力和 Android debug APK 构建能力；已实现角色卡导入 / 导出、私聊、群聊、动态、评论、生活事件、关系状态、时间线、今日简报、主动消息、写卡工具、提示词预设、本地备份等核心模块。当前阶段适合进入早期用户试用、产品打磨、正式发布准备和商业化验证。

## 2. 一句话定位

PalTavern 让 AI 角色卡变成长期陪伴的社交联系人和世界居民，而不只是聊天框里的回复对象。

更完整地说：

PalTavern 是面向角色卡用户、AI 伴侣用户和原创角色创作者的本地优先 AI 角色社交应用。它提供私聊、群聊、动态、事件、关系记忆和角色创作工具，让用户能够经营一个可长期积累的角色社交世界。

## 3. 用户痛点

### 3.1 AI 角色聊天仍然太像“单次会话”

很多角色聊天产品的主要交互仍然是用户发一句、角色回一句。它适合短期对话，却很难承载长期关系、共同事件、多角色互动和世界连续性。用户越投入，越容易遇到角色遗忘、上下文混乱、关系进展不可控等问题。

### 3.2 角色卡生态强大，但体验分散

SillyTavern 及相关角色卡生态已经沉淀了大量高自由度角色卡、世界书和提示词工作流。问题是，这些内容更像创作材料和聊天配置，缺少一个普通用户更熟悉的“联系人 + 动态 + 事件 + 记忆”的产品外壳。

### 3.3 多角色世界缺少社交结构

很多用户并不只和一个角色互动。他们会拥有乐队、宿舍、班级、家族、社团或原创世界观中的多个角色。传统聊天工具很难自然表达角色之间的低频互动、群聊接话、动态可见范围、共同事件和关系变化。

### 3.4 长期档案需要安全阀

长期角色体验最怕一次错误生成污染后续全部剧情。用户需要知道“为什么角色这样说”“这件事是否进入长期记忆”“不满意的事件影响能否撤销”。没有这些控制，自动生成越多，用户越不敢长期使用。

### 3.5 用户希望保留数据主权

核心用户往往在意角色卡、聊天记录、世界书、API Key 和本地创作素材的控制权。纯云端产品虽然上手简单，但也带来迁移困难、内容审查、数据锁定和隐私担忧。PalTavern 的本地优先路线正好服务这类高投入用户。

## 4. 产品方案

PalTavern 的核心方案，是把 AI 角色从“单个聊天对象”组织成“一个本地社交世界”。

### 4.1 角色成为联系人

用户可以导入 SillyTavern 角色卡，或者用内置 Character Studio 创建新角色。每个角色拥有独立私聊、头像、关系阶段、好感度、关系摘要、主动消息设置、自动动态设置、表情包和角色世界书。

### 4.2 私聊像真实通讯工具

PalTavern 的私聊不是简单套壳聊天框。它会结合角色设定、世界书、用户人设、关系状态、近期聊天、时间线、事件、天气、今日简报和表情包来生成更自然的手机聊天式回复，并尽量避免传统角色扮演中常见的大段舞台描写。

### 4.3 群聊承载多角色互动

用户可以创建群聊，让多个角色围绕同一话题自然接话，也可以指定某个角色说一句，或让用户以某个角色身份手写发言。群聊消息会进入世界时间线，成为后续上下文的一部分。

### 4.4 动态与评论形成轻量社交流

用户和角色都可以发布动态。动态支持公开、好友可见、指定角色、屏蔽角色、仅自己等可见范围。角色可以评论可见动态，角色之间也可以围绕动态产生低频互动。

### 4.5 生活事件推动世界变化

事件系统用于记录世界里发生的小变化。用户可以手写事件，也可以让模型生成生活线索，再选择后续分支或手写结果。事件可以影响角色关系，并写入时间线。

### 4.6 时间线沉淀长期记忆

时间线记录聊天记忆、群聊记忆、动态、评论、事件、关系变化、主动消息、今日简报、角色状态和用户手动记录。它既是用户回顾世界历史的界面，也是模型生成时读取长期上下文的安全入口。

### 4.7 关系系统让角色变化可见

每个角色有独立关系阶段、好感度和关系摘要。关系数据会影响私聊、群聊、动态、评论、事件和主动消息，让角色不只是静态设定，而是有当前关系处境。

### 4.8 主动消息带来存在感

角色可以在合适时间主动联系用户，并遵守每日上限、安静时段、随机间隔和未回复降频规则。用户连续不回复时，应用不会让角色机械刷屏；用户重新回复后，可以选择是否恢复正常频率。

### 4.9 Character Studio 降低写卡门槛

PalTavern 内置写卡工具，可以从一个想法逐步生成角色卡草稿。简单版适合快速成卡，复杂版适合长期角色，支持性格调色盘、二次解释、first_mes 生成、草稿管理、导出酒馆卡和直接创建联系人。

### 4.10 本地备份与模型可插拔

PalTavern 不内置大模型，用户配置兼容 OpenAI Chat Completions 的接口即可使用。数据主要保存在本机，支持完整备份 JSON 导入导出，角色也可以导出为 SillyTavern 角色卡。

## 5. 当前进展

PalTavern 当前不是纯概念项目，已经具备可运行 MVP。

已完成或已接入的能力包括：

- 本地 Web 应用主界面。
- Windows 桌面应用运行与托盘驻留能力。
- Android Capacitor 工程与 debug APK 构建能力。
- SillyTavern JSON / PNG 角色卡导入。
- 多角色卡候选识别。
- 角色卡导出。
- 私聊、消息编辑、撤回、删除、重新生成和多版本回复。
- 群聊、成员管理、指定角色发言和群聊时间线记录。
- 动态发布、角色生成动态、评论、回复和删除。
- 动态可见范围。
- 生活事件创建、生成、分支结算和关系影响。
- 世界时间线、手动记录、今日简报和角色状态摘要。
- 关系阶段、好感度和关系摘要。
- 主动消息、自动动态、自动事件和未回复降频。
- 通知权限、测试通知和后台调度准备。
- 表情包系统。
- 提示词预设导入与管理。
- Character Studio 写卡工具。
- 本地完整备份导入导出。

当前边界也很清楚：

- 还不是正式商店发布应用。
- Android 当前以 debug APK 测试包为主。
- 尚无云端账号和多设备实时同步。
- 后台主动消息受系统、省电策略和应用运行状态影响，不能承诺像即时通讯一样实时。
- 模型能力依赖用户配置的第三方或自定义接口。
- 当前不应承诺语音通话、真实文件发送、定位、闹钟、现实提醒等能力。

## 6. 目标用户

### 6.1 种子用户

PalTavern 的第一批用户应从高投入角色卡用户中获取：

- SillyTavern 中文用户。
- AI 角色扮演和长期陪伴用户。
- 原创角色、OC、世界观创作者。
- 喜欢收集、修改、经营角色卡的用户。
- 希望本地保存角色、聊天和创作资料的用户。

这些用户对功能复杂度的容忍度更高，也更愿意反馈角色卡兼容、提示词、长期记忆和多角色互动问题。

### 6.2 可扩展用户

在核心体验稳定后，PalTavern 可以向更泛化人群扩展：

- 想要低门槛创建 AI 角色的普通用户。
- 需要角色陪伴但不想被云端平台锁定的用户。
- 轻度文字创作者、同人创作者、剧情互动爱好者。
- 有角色 IP、虚拟主播、企划设定的创作者团队。
- 希望把角色卡变成可持续互动内容的内容运营者。

## 7. 市场判断

AI 角色产品正在从“单次聊天”向“长期关系”和“可运营角色资产”演进。未来的竞争不只是谁的模型更会说话，而是谁能让角色形成稳定身份、长期记忆、可控关系和可迁移资产。

PalTavern 押注的方向是：用户会越来越重视自己拥有的角色、设定、聊天记录、关系进展和世界记忆。这些内容不应该只被锁在单一平台的服务器里，而应该能在用户掌控的本地和开放格式之间流动。

在这个判断下，PalTavern 的切入点不是正面复制通用 AI 陪伴产品，而是先服务角色卡生态中的高价值用户，再把成熟能力包装给更大众的角色陪伴和原创角色市场。

本文未引用外部市场规模数字。若用于正式融资材料，建议后续补充经过来源核验的 AI 伴侣、虚拟角色、UGC 创作工具和二次元内容消费相关市场数据。

## 8. 差异化优势

### 8.1 从聊天框到社交世界

PalTavern 的核心差异不是“多一个 AI 聊天软件”，而是把角色组织成联系人、群聊成员、动态发布者和事件参与者。它更接近角色社交网络，而不是单条对话窗口。

### 8.2 兼容已有角色卡生态

PalTavern 支持 SillyTavern 角色卡导入导出，能承接用户已有的卡片、世界书和提示词习惯。这降低了冷启动成本，也让产品能借力已有创作生态。

### 8.3 本地优先与用户数据主权

PalTavern 当前以本地存储和本地备份为基础。对高投入创作者来说，角色设定、关系记录和聊天历史本身就是资产。本地优先能够建立信任，也为未来可选云同步提供空间。

### 8.4 模型可插拔

PalTavern 不把自己绑定到单一模型厂商。用户可以配置兼容接口，产品价值沉淀在角色组织、记忆结构、交互体验和创作工具层，而不是单点模型调用。

### 8.5 长期档安全机制

时间线、影响记录、可见范围、关系摘要和回滚思路，让用户能控制哪些内容进入长期记忆。这对长期角色关系非常关键，也是普通聊天产品常常缺失的一层。

### 8.6 创作与陪伴闭环

PalTavern 同时支持写卡、导入、私聊、群聊、动态、事件、备份和导出。用户可以从一个角色想法开始，逐步进入长期陪伴和世界经营，而不是在多个工具之间切换。

## 9. 商业模式设想

PalTavern 可以采用分阶段商业化，不急于在 MVP 阶段引入重云服务成本。

### 9.1 第一阶段：付费本地应用 / Pro 功能

基础版本用于获取用户和验证留存，高级版本提供更完整的长期使用能力，例如：

- 更高级的多世界管理。
- 更强的时间线搜索和筛选。
- 更完善的角色状态页。
- 更高级的备份、加密和迁移工具。
- 更完整的桌面 / Android 打包版本。
- 更多 Character Studio 写卡辅助能力。

付费方式可以是买断制、早鸟授权或会员制，需要在用户测试后再决定。

### 9.2 第二阶段：可选云服务

在本地优先的基础上，提供用户自愿开启的云能力：

- 多设备同步。
- 加密云备份。
- 角色世界跨设备迁移。
- 远程通知和后台调度增强。

这部分可以形成订阅收入，但必须明确隐私边界，不能破坏本地优先用户的信任。

### 9.3 第三阶段：创作者生态与内容市场

当用户和创作者规模足够后，可以围绕角色卡和世界模板建立内容生态：

- 角色卡模板。
- 世界观模板。
- 提示词预设。
- 表情包和界面主题。
- 角色关系包。
- 官方精选或创作者分成市场。

PalTavern 在这里可以做工具、分发和交易基础设施，而不是只卖单个软件授权。

### 9.4 第四阶段：IP 与创作者合作

PalTavern 可以为虚拟主播、原创企划、同人团队和小型 IP 方提供“角色社交空间”解决方案，让角色不只是发布内容，而是能以联系人、动态和事件的方式持续陪伴粉丝。

## 10. 增长路径

### 10.1 从核心社区切入

第一阶段应优先服务 SillyTavern 和角色卡核心用户。原因是他们已有角色资产、理解模型配置、愿意测试复杂功能，也能给出高质量反馈。

推荐动作：

- 发布可安装测试包和清晰的新手文档。
- 用真实角色卡展示从导入到私聊、群聊、动态、事件、时间线的完整流程。
- 以“让你的角色像联系人一样生活”为核心传播信息。
- 鼓励用户分享自己的角色世界截图、动态流和群聊片段。

### 10.2 用创作者拉动内容

角色卡创作者和原创角色作者是 PalTavern 的关键增长节点。只要工具能让他们更容易展示角色、经营关系和产出可分享片段，就可能形成自然传播。

推荐动作：

- 提供写卡教程和 Character Studio 模板。
- 邀请创作者制作示例角色世界。
- 建立精选角色卡、世界模板和使用案例。
- 让创作者能清楚知道自己的卡在 PalTavern 中能获得哪些新增体验。

### 10.3 逐步降低普通用户门槛

当核心功能稳定后，需要把“配置模型、导入角色、设置人设、备份数据”这些步骤做成更低门槛的新手引导。普通用户不应该先理解技术概念，应该先体验到角色像联系人一样存在。

## 13. 风险与应对

### 13.1 市场过窄风险

PalTavern 从 SillyTavern 用户切入，早期人群相对垂直。应对方式是先在高强度用户中验证长期留存和付费意愿，再把成熟能力包装成更低门槛的普通角色陪伴工具。

### 13.2 模型依赖风险

产品体验依赖第三方模型质量和成本。应对方式是保持模型可插拔，让 PalTavern 的核心价值沉淀在角色组织、记忆结构、社交交互和创作工具，而不是绑定单一模型。

### 13.3 内容安全和平台政策风险

用户生成角色和对话可能涉及成人、敏感或平台限制内容。应对方式是明确本地数据边界、应用分发限制、年龄提示、内容导入责任和可选安全设置，并在正式分发前补齐合规文档。

### 13.4 隐私和数据安全风险

本地备份可能包含 API Key、角色设定和聊天记录。应对方式是提供清晰提示、备份加密、导入前确认、敏感字段说明和最小化云端依赖。

### 13.5 产品复杂度风险

PalTavern 功能较多，普通用户可能被设置项吓退。应对方式是保留高级能力，但用默认配置、新手引导、推荐流程和渐进式设置降低首次使用负担。

## 14. 当前适合对投资人表达的结论

PalTavern 当前最有价值的不是一个单点功能，而是一套已经跑通的产品结构：

- 它承接已有角色卡生态。
- 它把角色从聊天框升级为社交联系人。
- 它让多角色、动态、事件和时间线形成长期世界。
- 它保留用户对本地数据和模型选择的控制权。
- 它已经具备可运行 MVP，而不是停留在概念阶段。

下一阶段最重要的任务，是把这个 MVP 打磨成可公开试用、可形成早期留存、可验证付费意愿的产品版本。

## 15. 对外简版介绍

PalTavern 是一个面向 AI 角色卡用户的本地优先角色社交应用。它可以导入或创建角色卡，让角色成为独立联系人，和用户私聊、进入群聊、发布动态、评论、参与生活事件，并通过关系状态、时间线和今日简报沉淀长期记忆。PalTavern 的目标不是再做一个普通 AI 聊天框，而是让用户经营一个由角色、关系和日常事件组成的私人 AI 社交世界。
