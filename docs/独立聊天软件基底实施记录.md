# 独立聊天软件基底实施记录

本文档记录“先做独立基底，再继续主动消息等功能”的当前实现结果。它是后续恢复上下文时优先阅读的主线文档。

## 目标修正

本项目现在的主线不是“小手机脚本”，而是真正独立的 AI 社交聊天应用。

当前约束：

- 不依赖 SillyTavern shim。
- 不加载小手机原版 bundle。
- 不以小手机脚本作为核心数据源。
- 不复用小手机适配层源码；只参考已经讨论过的功能目标重新写独立应用。
- 保留 Electron 桌面版和 Capacitor Android 打包能力。
- 先完成角色卡导入、通讯录、私聊、模型配置和本地持久化。
- 后续再接主动消息、后台通知、动态、世界、关系和导出回酒馆等功能。

本次同时补齐：

- `index.ts`
  - 只负责加载样式并启动界面渲染。
- `types.ts`
  - 记录独立应用自己的数据结构。
  - 包含世界、角色、会话、消息、动态、模型配置和主动消息节奏结构。
  - 新增 `RelationshipState`、`RelationshipStage` 和 `ModelUsage`。
- `utils.ts`
  - 放通用工具，例如 ID、稳定哈希、HTML 转义、文本压缩。
- `state.ts`
  - 负责默认状态、本地读取、本地保存、当前世界、当前角色和会话创建。
  - 这里也负责把旧的本地状态迁移成新的独立结构。
  - 支持创建、切换和删除世界。
  - 删除世界时同步清理该世界的角色、会话、消息和动态。
- `backup.ts`
  - 负责本地手动备份。
  - 可导出当前世界、角色、聊天、动态、模型配置和主动消息节奏。
  - 可导入备份并替换当前本地状态。
  - 导入时会先规范化状态结构，兼容旧字段。
- `cards.ts`
  - 负责解析 JSON / PNG 角色卡并写入独立应用自己的角色模型。
  - 只读取角色卡功能字段，不复用旧小手机源码。
- `tavern-export.ts`
  - 把当前角色导出为 SillyTavern V2 JSON 角色卡。
  - 保留原角色卡的未知顶层字段、作者字段和第三方扩展。
  - 使用当前角色资料覆盖旧卡中的名称、描述、性格、场景、开场白、标签和世界书。
  - 关系状态写入 `data.extensions.tavern_social`。
  - 不导出 API Key、聊天记录、模型配置、主动消息规则或调度状态。
- `png-card-parser.mjs`
  - 负责读取 PNG 的 `tEXt` / 未压缩 `iTXt` 文本块。
  - 支持 `chara` 和 `ccv3` 元数据键。
  - 支持 base64 UTF-8 角色卡内容。
  - 作为纯模块可在 Node 中单独构造 PNG 数据测试。
- `model.ts`
  - 负责 OpenAI-compatible 消息组装和模型请求。
  - 关系阶段、亲密度和关系摘要进入系统提示。
  - 所有真实模型请求遵守每日请求预算。
- `chat.ts`
  - 负责用户发送消息、写入聊天记录、调用模型、记录用户回复时间。
  - 用户回复后只标记 `pendingResetDecision`，不自动恢复主动消息频率。
- `moments.ts`
  - 负责读取当前世界的动态流。
  - 支持用户或当前世界中的角色发布动态。
  - 支持调用当前 OpenAI-compatible 模型生成角色动态。
  - 支持删除动态，并保证不同世界的动态互相隔离。
- `events.ts`
  - 负责按世界保存生活事件。
  - 事件可指定一个或多个参与角色。
  - 创建事件时可对参与角色应用 `-20` 到 `20` 的关系影响。
  - 事件可标记为进行中或已结束，也可删除。
  - 支持让当前角色调用模型生成一条生活事件。
  - 删除事件不会自动回滚已经产生的关系变化，避免历史状态突然逆转。
- `notifications.ts`
  - 负责独立应用的本地通知基础。
  - 当前只提供通知权限状态、申请权限、通知文本隐私处理和测试通知。
  - 不承担后台调度；后台调度会在后续 Windows 托盘和 Android 系统任务阶段接入。
- `platform.ts`
  - 负责识别当前运行宿主：普通 Web、Electron 桌面、Android、iOS。
  - 负责显示后台宿主能力：仅运行期检查或可后台驻留。
  - 负责定义未来 Android 原生后台桥契约。
  - 如果没有 Android 原生桥，Web 端不会假装后台任务已经可用。
- `scheduler.ts`
  - 负责独立应用内的主动消息调度核心。
  - 调度只在应用实际运行时检查。
  - Windows 桌面版最小化到托盘时，渲染进程仍保持运行，因此这个调度器可以继续检查。
  - 启动时如果发现 `nextAttemptAt` 已经过期，会跳过错过的尝试并安排下一次，不补算、不补发。
  - 到达尝试时间后会检查安静时段、每日主动消息上限、模型配置和节奏状态。
  - 主动消息写入私聊，消息来源标记为 `auto_message`。
  - 未回复时根据角色性格和未回复次数拉长下一次间隔。
  - 用户回复后只标记 `pendingResetDecision`，等待用户选择恢复或保持降频。
  - 关系阶段会微调不同性格角色的降频倍率。
  - 每次主动生成前检查全局模型请求预算。
- `ui.ts`
  - 负责渲染通讯录、聊天窗口、设置面板和主动消息骨架预览。

这次拆分同时修复了早期 `index.ts` 中出现的中文编码损坏问题。当前独立目录已重新写成可读中文文本。

## 当前独立基底功能

已经完成：

- 独立 UI 壳：左侧通讯录，中间私聊，右侧世界、模型和角色信息。
- 本地状态存储：使用 `localStorage` 保存角色、会话、消息、用户名称和模型配置。
- 本地手动备份：
  - 导出 `tavern-social-backup-*.json`。
  - 导入备份 JSON 后替换当前本地状态。
  - 备份包含世界、角色、聊天、动态、模型配置和主动消息节奏。
- 独立数据模型：
  - `WorldProfile`
  - `CharacterProfile`
  - `ConversationProfile`
  - `ChatMessage`
  - `MomentEntry`
  - `ModelConfig`
  - `AppState`
- 主动消息数据与运行调度：
  - `AutoMessageSchedule`
  - `NotificationPrivacy`
  - `PacingState`
  - `QuietHours`
  - 已经接入应用运行期间的调度核心。
  - Windows 托盘驻留时渲染进程保持运行，主动消息检查继续执行。
  - Android 系统后台任务仍未完成。
- JSON / PNG 角色卡导入：
  - 读取常见字段：`name`、`description`、`personality`、`scenario`、`first_mes`、`tags`、`avatar`。
  - 保留原始卡片数据到 `rawCard`。
  - 如果角色卡有开场白，会自动写入该角色的私聊。
  - 角色 ID 包含世界维度，同一张角色卡可以分别导入不同世界，不会互相覆盖。
  - PNG 卡从标准 `chara` / `ccv3` 文本块读取元数据。
  - PNG 图片会生成最大 512 像素的 WebP 头像缩略图，避免把整张大图直接写入本地存储。
  - 记录导入来源格式、文件名、卡片规范、规范版本和内嵌世界书条目数量。
  - 支持读取 `character_book`。
  - 常驻世界书条目会加入模型上下文。
  - 关键词世界书条目按近期聊天命中后加入模型上下文。
  - 世界书上下文设置 6000 字符上限，避免无限膨胀。
- 导出回酒馆：
  - 当前角色信息区提供“导出酒馆角色卡 JSON”。
  - 导出格式固定为 `chara_card_v2` / `2.0`。
  - 原卡第三方字段和扩展不会因为独立应用编辑而被无条件丢弃。
  - 文件名根据角色名称生成，并过滤 Windows 不允许的文件名字符。
  - 独立应用的私密配置不会进入角色卡。
- 多世界管理：
  - 用户可以创建新世界。
  - 用户可以在世界下拉框中切换当前世界。
  - 通讯录只显示当前世界中的角色。
  - 私聊、消息、动态和主动消息状态按世界隔离。
  - 可以删除非最后一个世界；删除前需要确认。
- 世界动态：
  - 顶部可在“私聊”和“动态”两个视图之间切换。
  - 动态发布者可以选择用户本人或当前世界中的任一角色。
  - 用户可以直接编写动态。
  - 角色可以依据角色设定和世界背景调用模型生成动态。
  - 动态只出现在所属世界中，切换世界后显示对应世界的动态流。
  - 支持删除已有动态。
- 世界事件与生活模拟：
  - 顶部新增“事件”视图。
  - 用户可填写事件标题、事件正文、参与角色和关系影响。
  - 没有参与角色时，事件作为整个世界的全局事件保存。
  - 有参与角色时，角色亲密度和关系摘要会立即更新。
  - 当前世界最近 5 条相关事件进入角色模型上下文。
  - 事件可标记结束，模型上下文会区分“进行中”和“已结束”。
  - 事件按世界隔离，删除世界时同步删除该世界事件。
  - 本地备份自动包含世界事件。
- 私聊：
  - 每个角色拥有独立会话。
  - 用户发消息后写入当前角色会话。
  - 未配置模型时返回本地占位回复，便于先验证界面流程。
- OpenAI-compatible 模型配置：
  - API 地址。
  - API Key。
  - 模型名。
  - 温度。
  - 每日模型请求预算。
  - 当日请求用量。
- 角色关系：
  - 关系阶段：陌生、熟悉、亲近、亲密、紧张。
  - 亲密度范围为 `-100` 到 `100`。
  - 可记录自由文本关系摘要。
  - 关系状态进入聊天、动态和主动消息的模型上下文。
- 本地通知：
  - 显示当前通知权限状态。
  - 申请通知权限。
  - 按当前角色的通知隐私等级发送测试通知。
  - 主动消息成功写入私聊后会按角色隐私设置发送通知。
  - 通知构造失败不会中断消息入库或其他角色的调度。
- 后台宿主状态：
  - 普通 Web 环境显示为仅运行期检查。
  - Electron 桌面环境显示为可后台驻留。
  - Android 环境需要 `window.TavernSocialAndroidBackground` 原生桥。
  - 没有原生桥时，Android 不会伪装成后台调度已完成。
- 主动消息运行期调度：
  - 每个角色可独立启用或关闭。
  - 可设置基础随机间隔、安静时段、每日上限、最大间隔、后台通知和通知隐私。
  - 可手动触发一次调度检查。
  - 调度器不会在应用重开后补发错过的消息。
  - 用户回复后出现恢复确认：恢复正常频率或保持当前降频。
- 独立模型代理：
  - `scripts/independent-chat-server.mjs`
  - `scripts/serve-independent-chat.mjs`
  - 默认接口路径为 `/api/chat-completions`。

## 主动消息原则备份

后续实现主动消息时继续遵守这些原则：

- 不做离线补算。
- 不做重开后补发。
- 只有后台调度实际运行成功时才生成消息。
- 生成成功后写入对应私聊并发送本地通知。
- Android 后台任务不是强实时承诺，系统限制时安全跳过。
- 用户长期不回复时，同一角色主动消息间隔逐渐变长。
- 不同角色根据性格表现不同降频行为。
- 用户回复后不自动恢复频率，等待用户选择或保持当前节奏。
- 所有主动消息遵守安静时段、每日上限、预算上限和通知隐私设置。
